┌───────────────────────────────────────────────────────────────┐
│  📦 STEP 1: Build (Optional - Test Locally)                   │
└───────────────────────────────────────────────────────────────┘

  npm install
  npm run build
  npx vite preview    # Test at http://localhost:4173


┌───────────────────────────────────────────────────────────────┐
│  🔄 STEP 2: Push to GitHub                                    │
└───────────────────────────────────────────────────────────────┘

  git add .
  git commit -m "Ready for deployment"
  git push origin main


┌───────────────────────────────────────────────────────────────┐
│  ☁️  STEP 3: Deploy to Vercel                                 │
└───────────────────────────────────────────────────────────────┘

  Option A: Vercel Dashboard (Recommended)
  ─────────────────────────────────────────
  1. Go to https://vercel.com
  2. Click "Add New Project"
  3. Import your repository
  4. Click "Deploy"
  ✅ DONE! Live in 2 minutes!

  Option B: Vercel CLI (Fastest)
  ─────────────────────────────────────────
  npm install -g vercel
  vercel --prod
  ✅ DONE!

╔═══════════════════════════════════════════════════════════════╗
║              🔐 Environment Variables (Optional)              ║
╚═══════════════════════════════════════════════════════════════╝

  Your Firebase config has fallback values, so it works without
  environment variables. For production security:

  1. Go to Vercel → Your Project → Settings → Environment Variables
  2. Add the variables from .env.example
  3. Redeploy
